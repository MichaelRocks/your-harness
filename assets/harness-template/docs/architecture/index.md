# Architecture Docs

Use this area for durable information about system structure, ownership,
boundaries, topology, and dependency rules.

## Start Here

- `docs/architecture/system-map-template.md`
- `docs/architecture/boundaries.md`
- `docs/architecture/dependency-rules.md`

## What Belongs Here

- High-level component maps
- Service, module, or package boundaries
- Allowed dependency directions
- Ownership and integration surfaces
- Runtime topology that affects implementation and operations

## What Does Not Belong Here

- Transient debugging notes
- One-off implementation details for a single change
- Behavior specs that should live under `docs/specs/`
