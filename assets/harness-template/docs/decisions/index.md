# Decisions

Use this area for durable cross-cutting decisions that should outlive a single
change plan.

## When To Write a Decision Record

- The choice affects multiple parts of the system or team workflow
- The tradeoff is non-obvious and worth preserving
- Future contributors are likely to revisit the same question

## Naming

- Use `YYYY-MM-DD-short-title.md` for decision records.
- Link related plans, specs, and architecture docs.
