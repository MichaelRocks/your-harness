# Tech Debt Log

Capture meaningful debt discovered during implementation but intentionally left
out of the current change.

## Entry Template

```md
### Short title

- Source: `docs/plans/completed/YYYY-MM-DD-short-slug.md`
- Area:
- Problem:
- Risk if left as-is:
- Why it was deferred:
- Suggested follow-up:
- Prevention benefit:
- Owner:
- Status: `open` | `planned` | `done`
```

## Rules

- Do not use this file to hide unfinished critical work that should block the
  current change.
- Link back to the originating completed plan whenever possible.
- Prefer a small number of clear debt items over a vague backlog dump.
