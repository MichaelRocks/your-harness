# Specs

Use this area for behavior contracts and end-to-end journeys. The goal is to
anchor implementation and testing to observable outcomes.

## Start Here

- `docs/specs/behavior-template.md`
- `docs/specs/journey-template.md`

## Rules

- Describe what users, systems, or operators can observe.
- Include success cases, failure cases, and telemetry expectations.
- Link specs to plans, tests, and decisions where practical.
