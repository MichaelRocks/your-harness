# Behavior Spec Template

Use this when defining or updating an observable contract.

## Template

```md
# Behavior name

## Intent

What capability this behavior provides and for whom.

## Trigger

How the behavior starts: user action, API call, event, schedule, deployment,
or operator action.

## Observable Result

- Expected visible output, state change, or side effect
- Error behavior and user/operator-visible messages

## Acceptance Criteria

- Concrete externally visible criterion

## Telemetry Expectations

- Logs:
- Metrics:
- Traces:
- Alerts or dashboards:

## Test Coverage Plan

- Behavior-level automated test:
- Supporting lower-level tests:
- Manual or operational validation:
```
